# Babio V2: вердикт и план

## 1. Вердикт: что такое V2

**V2 = enriched product prototype + recording emulator. Не production app.**

Это не компромисс «на всякий случай», а осознанный выбор:

| Роль | Зачем |
|------|--------|
| **Recording emulator** | Paid social, демо инвесторам/партнёрам, контролируемый narrative |
| **Product prototype** | Проверить, возвращаются ли мамы, доверяют ли «одному спокойному шагу», покрывают ли сценарии реальную неделю |
| **Не production** | Нет backend, auth, real AI, медицинской ответственности, retention-инфраструктуры |

**Почему не production:** позиционирование Babio строится на доверии и безопасности. Production без педиатрического advisory, compliance review и human-in-the-loop для edge cases — преждевременный риск. Сейчас вы продаёте **обещание спокойствия и ясности**, а не медицинский сервис.

**Почему не «только emulator»:** пять flows уже исчерпали демо-формат. Для валидации продукта нужны: повторное использование (log → ask), персонализация по возрасту, ощущение «приложение знает мою ситуацию» — без этого screencast выглядит как красивый one-shot, а не продукт.

## 2. Top 5 product capabilities для V2

### 1. Baby profile + age-aware context
Имя, дата рождения (или «N недель»), режим кормления (грудь/смесь/смешанное), сон (колыбель/кроватка/совместный).

**Зачем:** без этого «персонализированный next step» — декорация. В каждом flow и в home должно быть видно: «для Маши, 11 недель».

### 2. Persistent session state (localStorage, без backend)
Log, notes, последний ask, «сегодняшний brief», dismissed tips.

**Зачем:** screencast «мама вернулась через 3 дня» и product test «я открыла снова — всё на месте».

### 3. Ask → structured answer template
Не чат, а **guided response**: «что мы учли» → «что можно попробовать» → «на что обратить внимание» → «что записать» → «когда обратиться к педиатру».

**Зачем:** это ядро value prop и главный safety guardrail — формат фиксирует тон и границы.

### 4. Log → Ask bridge
Из записи в log («плохой сон 2:40», «кормление 15 мин») — one-tap «спросить про это».

**Зачем:** проверяет гипотезу «контекст из жизни → один шаг», а не «ещё один FAQ».

### 5. Scenario library + studio improvements
12–15 новых flows, теги (ночь/кормление/здоровье), pre-filled baby state для record mode, варианты CTA для A/B в креативах.

**Зачем:** emulator остаётся движком дистрибуции; библиотека сценариев — движок валидации.

## 3. 12–15 дополнительных safe scenarios (по user moment)

### Ночь и сон
1. **«Проснулась через 45 минут после укладывания»** — что попробовать сейчас, что отслеживать 3 ночи, не «исправим сон».
2. **«Дневной сон: 20 минут и проснулась»** — возрастная норма vs паттерн, что записать.
3. **«Регресс / всё сломалось на этой неделе»** — validation + 1–2 безопасных эксперимента, без диагноза «регресс 4 месяцев».

### Кормление
4. **«Кажется, мало ест / часто просит»** — checklist наблюдений (мокрые подгузники, вес на приёме), не «недокорм».
5. **«Отрыжка / срыгивания после каждого кормления»** — что нормально, что отслеживать, когда звонить.
6. **«Переход на смесь / смешанное»** — practical next steps, без brand recommendations как medical advice.

### Тело и комфорт
7. **«Сыпь на щеках — это нормально?»** — observation framing, фото для педиатра, red flags без страшилок.
8. **«Животик крутит / плачет после кормления»** — что попробовать (поза, пауза), что логировать.
9. **«Температура 37.8 — что делать сейчас»** — чёткий «когда звонить», без самолечения и без «всё ок».

### Рутина и усталость мамы
10. **«Утренний brief: как прошла ночь»** — 60 сек summary + один микро-шаг на день.
11. **«Я устала и сомневаюсь во всём»** — emotional validation + один actionable step, не терапия.
12. **«Партнёр/бабушка говорит иначе»** — как сформулировать вопрос педиатру, не «кто прав».

### Визиты и подготовка
13. **«Приём через 2 дня — что собрать»** — export из log/notes в «лист для педиатра».
14. **«Новый симптом с утра — спросить или подождать»** — decision tree с conservative default «если сомневаешься — спроси».

### Развитие (без milestone anxiety)
15. **«14 недель — что имеет смысл наблюдать»** — ranges, не «должна уметь», с акцентом «спроси, если беспокоит».

## 4. Что НЕ добавлять в V2 (и почему)

| Не добавлять | Почему |
|--------------|--------|
| Real AI / LLM chat | Непредсказуемый тон, галлюцинации, compliance risk; ломает record mode |
| Auth, accounts, push | Не нужно для screencast и hallway test; отвлекает от product hypothesis |
| Community / forums | Модерация, медицинский misinformation, другой продукт |
| Diagnosis labels («колики», «РПП») | Юридически и этически опасно без клинического oversight |
| Medication dosing / «что дать» | Прямой medical advice territory |
| Growth charts / percentile anxiety | Fear hooks; против positioning «спокойный шаг» |
| Subscription / paywall | Рано; сначала moment coverage и trust |
| Native iOS/Android | Emulator на web достаточен для validation и ads |
| Multilingual | Copy explosion; русский/английский — отдельный этап после message-market fit |
| Integrations (Apple Health, wearables) | Engineering rabbit hole, слабая связь с core loop |

## 5. Риски

### Compliance / trust
- Любая формулировка «это нормально» без «если X — обратись» читается как медицинское заверение.
- Screencast для ads может вырезать disclaimers — нужен **on-screen persistent micro-disclaimer** в record mode.
- «Персонализация» без реальной логики создаёт **ложное ощущение expertise**.

**Митигация:** scripted templates, mandatory red-flag block, legal review 10–15 эталонных ответов до масштабирования copy.

### UX
- Слишком много flows → studio становится неуправляемым.
- Log без структуры → мама не понимает, зачем логировать.
- Ask без контекста baby profile → разрушает trust за один тап.

**Митигация:** один golden path (profile → log → ask → note), остальное — библиотека.

### Product scope
- Риск стать «baby tracker №847» вместо «conflicting advice → one calm step».
- Риск уйти в sleep coaching / feeding coaching — другие конкуренты, другие ожидания.

**Митигация:** каждый новый flow проходит фильтр: *«какой conflicting advice мы снимаем?»*

### Engineering
- Data-driven copy в `babio.ts` раздуется → нужна **schema** (flow, steps, variables, disclaimers).
- localStorage без versioning сломает demos после обновлений.
- Record mode и manual mode расходятся → двойная поддержка.

**Митигация:** единый flow definition, `?demo=` presets, migration v1→v2 для localStorage.

## 6. Этапы реализации с acceptance criteria

### Stage 0 — Foundation (1 sprint)
**Сделать:** baby profile, localStorage session, age calculation, disclaimer component, flow schema refactor.

**AC:**
- [ ] При первом входе — mini onboarding (имя + возраст), skip для studio.
- [ ] Home показывает «Привет, [имя] · [N недель]».
- [ ] Перезагрузка страницы сохраняет profile + log + notes.
- [ ] Каждый answer screen содержит блок «не заменяет педиатра».

---

### Stage 1 — Core loop (1–2 sprints)
**Сделать:** structured ask template, log → ask bridge, 3 новых flows (ночь, кормление, сыпь).

**AC:**
- [ ] Ask всегда ренерит 5 секций (учли / попробовать / наблюдать / записать / педиатр).
- [ ] Из log entry — кнопка «Спросить про это» с pre-filled context.
- [ ] 3 flows проходят manual + record mode без правки кода (только data).
- [ ] Screencast 30 сек: profile → log → ask → один calm step — снимается одним дублем.

---

### Stage 2 — Scenario library (1 sprint)
**Сделать:** ещё 9–12 flows, теги в studio, search/filter, demo presets.

**AC:**
- [ ] Studio: фильтр по moment (ночь, кормление, тело, рутина, визит).
- [ ] Record mode: `?preset=night-45min` подставляет baby + log state.
- [ ] ≥15 flows total, у каждого есть disclaimer variant.
- [ ] Ни один flow не содержит diagnose/promise language (lint по forbidden phrases).

---

### Stage 3 — Daily brief + pediatrician export (1 sprint)
**Сделать:** morning brief на основе log, «сохранить для педиатра» как собранный PDF/text block (client-side).

**AC:**
- [ ] Brief генерируется из последних 24h log entries (scripted, не AI).
- [ ] Export: дата, возраст, симптомы/паттерны, вопросы мамы — copy-to-clipboard.
- [ ] Flow `save-for-pediatrician` обновлён под новый export format.

---

### Stage 4 — Validation instrumentation (0.5 sprint)
**Сделать:** in-prototype analytics stubs (console/events), session replay markers для user tests.

**AC:**
- [ ] События: `profile_completed`, `log_entry`, `ask_completed`, `flow_recorded`.
- [ ] 5-минутный moderated test script + success metric: «поняла next step» ≥4/5.

**Не Stage 4 V2:** backend, auth, payments, real AI.

## 7. Первый vertical slice (строить следующим)

### Slice: **«Плохая ночь → log → ask → один шаг»**

Один сквозной путь, один новый flow, минимальные зависимости — но проверяет всю гипотезу Babio.

**Scope:**
1. Baby profile (имя + 9 недель) — onboarding или studio preset.
2. Log entry: «Проснулась в 2:40, бодрствование 45 мин» (quick tap + time).
3. Ask pre-filled: «Проснулась снова ночью».
4. Structured answer (scripted): учли возраст + запись в log → 2 «попробовать сейчас» → 2 «наблюдать 3 ночи» → «записать в notes» → «позвони, если…».
5. Record mode: autoplay этого пути за 25–35 сек для Reels/TikTok.

**Почему именно это:**
- Усиливает существующий `baby-woke-up-again`, но добавляет **log → ask bridge** — то, чего нет в чистом emulator.
- Ночь — самый сильный emotional moment для first-time moms и paid social.
- Полностью data-driven: можно написать copy, снять screencast и провести 5 user tests без backend.

**AC для slice (Definition of Done):**
- [ ] Новый пользователь (или preset) проходит путь за <90 сек в manual mode.
- [ ] Record mode выдаёт чистый green-screen ролик без ручных тапов.
- [ ] 3 независимых ревьюера (вы + 2 мамы/знакомых) говорят: «понятно, что делать дальше» и «не страшно».
- [ ] Нет формулировок diagnose/promise/fix в copy.

## Резюме в одной строке

**V2 Babio = тот же web emulator для ads, но с baby profile, persistent log, structured ask и 15+ scripted moments — достаточно, чтобы выглядеть как приложение и проверить trust, недостаточно, чтобы быть medical product.**

Если нужно, могу следующим сообщением расписать **конкретную schema для flows в TypeScript** или **shot list для первого screencast** под этот vertical slice — без инспекции репозитория, чисто как product spec.
